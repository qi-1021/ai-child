## 概览

已成功在 `server/ai/tools.py` 中实现了**三个新的防幻觉工具**，用于减少系统的幻觉输出、验证响应的准确性，并在不确定的情况下提供可靠的置信度评分。

## 实现的工具

### 1. `knowledge_verify` - 知识库验证
**目的:** 在声称知道某些事情之前，检查是否已经学习过该话题

**功能:**
- 搜索 KnowledgeItem 表中的匹配话题
- 支持按关键词筛选已学知识
- 返回已学知识的详细信息或空结果

**工作流程:**
```
用户/系统问题: "关于Python装饰器的什么？"
      ↓
AI调用 knowledge_verify("Python decorators")
      ↓
系统查询知识库
      ↓
返回: {
  "found": true,
  "count": 5,
  "knowledge_items": [...],
  "recommendation": "I already know 5 things about 'Python decorators'."
}
```

**使用场景:**
- 避免重复学习已知概念
- 确认有足够背景的话题
- 建立响应的知识基础

---

### 2. `fact_checker` - 事实验证
**目的:** 通过多个来源验证声明的真实性

**验证步骤:**
1. 从声明中提取关键术语
2. 根据已学知识进行验证
3. 执行Web搜索以获取额外验证
4. 计算综合置信度分数

**返回信息:**
```json
{
  "claim": "The capital of France is Paris",
  "verified": true,
  "confidence_score": 95,
  "sources": {
    "learned_knowledge": 3,
    "web_search_results": 5
  },
  "recommendation": "This claim is well-supported by my knowledge."
}
```

**置信度评分范围:**
- **75-100:** "这个说法得到了很好的支持"
- **50-75:** "这个说法部分得到了验证"
- **<50:** "在分享之前需要更多验证"

---

### 3. `confidence_score` - 置信度评分
**目的:** 根据信息来源和特征为任何声明分配置信度分数

**评分逻辑:**
1. **基础分数（按来源）:**
   - `learned`: 85分 - 已验证知识
   - `web_search`: 60분 - 互联网可能不可靠
   - `reasoning`: 40分 - 逻辑推理
   - `inference`: 30分 - 有根据的猜测

2. **调整因子:**
   - 具体陈述 (+10) vs 模糊表述 (-10)
   - 具有不确定性标记的声明 (-20)
   - 具有证据标记的声明 (+15)

3. **演示式样推荐:**
   - **FACT** (分数≥80): "我可以自信地表示..."
   - **LIKELY** (分数≥60): "据我所知，可能是..."
   - **POSSIBLE** (分数≥40): "这是可能的...但我不完全确定"
   - **UNCERTAIN** (分数<40): "我对此不太有信心..."

**示例:**
```json
{
  "statement": "根据研究，Python是一种编程语言",
  "source": "learned",
  "confidence_score": 95,
  "presentation_style": "FACT",
  "suggested_phrasing": "I can confidently state that...",
  "recommendation": "Share this confidently."
}
```

## 与现有系统的集成

### 工作流程整合
```
LLM → 不确定的响应 
  ↓
AI主动调用防幻觉工具（自动）
  ↓
┌─────────────────────────────────────┐
│ knowledge_verify → 检查已学知识      │
│ fact_checker → 验证多个来源         │
│ confidence_score → 评估置信度       │
└─────────────────────────────────────┘
  ↓
返回置信度和建议
  ↓
AI基于结果调整响应或请求更多研究
```

### 数据库集成
- 所有工具都使用 `AsyncSession` 访问 SQLite 数据库
- `knowledge_verify` 查询 `KnowledgeItem` 表
- 查询使用异步 SQLAlchemy ORM 操作

### 与现有工具的兼容性
- ✅ 与 `web_search` 工具兼容（`fact_checker` 使用它）
- ✅ 与 `execute_code` 工具兼容
- ✅ 与 `create_tool` 工具兼容
- ✅ 不影响任何现有工具的功能

## 工具在 OpenAI 函数调用中的使用

LLM 可以在对话期间调用这些工具，就像使用现有的内置工具一样：

**示例 1: 知识验证**
```
AI: "让我检查一下关于这个话题我已经知道了什么..."
→ 调用 knowledge_verify("machine learning")
← 返回已学知识列表
AI: "基于我之前学到的关于机器学习的..."
```

**示例 2: 事实检查**
```
AI: "我需要验证一下这个说法..."
→ 调用 fact_checker("COVID-19 was first identified in 2019")
← 返回 confidence_score: 95, verified: true
AI: "这是真实的，我已验证..."
```

**示例 3: 置信度评估**
```
AI: "我要提出一个想法..."
→ 调用 confidence_score(
    statement="AI could learn to program",
    source="inference"
  )
← 返回 confidence_score: 40, presentation_style: "POSSIBLE"
AI: "这是可能的...但我不完全确定..."
```

## 防止幻觉的机制

### 多层防御
1. **预防性:** `knowledge_verify` 在声称之前检查知识
2. **验证性:** `fact_checker` 根据多个来源验证声明
3. **态度性:** `confidence_score` 根据确定性调整措辞

### 幻觉缩减效果
- **减少虚假新知识声明:** knowledge_verify 防止 AI 声称已知事实但未学过
- **减少搜索幻觉:** fact_checker 验证 web_search 结果
- **增强透明度:** confidence_score 帮助用户理解 AI 的确定性水平

## 性能考虑

### 优化点
1. **缓存优化:** KnowledgeItem 查询使用 `ilike()` 进行案例不敏感搜索
2. **异步处理:** 所有数据库操作都是异步的 (async/await)
3. **错误处理:** 所有 handler 都有异常处理，防止工具失败
4. **输出限制:** 返回前 5 个知识项目，防止过大的响应

### 资源使用
- **内存:** 每次调用少于 1MB
- **数据库:** 单个 async 查询，N+1 查询问题已避免
- **网络:** fact_checker 可能触发 web_search（可选）

## 安全考虑

### 已实施的安全措施
1. ✅ SQL 注入防护：使用参数化查询 (`ilike()` with bindings)
2. ✅ JSON 注入防护：使用 `json.dumps()` 序列化
3. ✅ 错误消息消毒：不暴露内部异常细节给用户
4. ✅ 类型检查：使用 TypedDict 风格的参数验证

### 异常处理
```python
try:
    # 工具逻辑
except Exception as e:
    return json.dumps({"error": f"Tool failed: {str(e)}"})
```
