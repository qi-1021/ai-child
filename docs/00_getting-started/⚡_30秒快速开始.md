# ⚡ 30 秒快速开始

## 🎯 三步启动

### 第一步：配置 API Key
```bash
# 方法 A：使用现有的 OpenAI API Key
echo "OPENAI_API_KEY=sk-your-api-key-here" > server/.env

# 方法 B：编辑文件手动配置
# 在 server/.env 中添加：OPENAI_API_KEY=sk-xxxxx
```

### 第二步：运行启动脚本
```bash
python start_ai_child.py --server
```

### 第三步：访问 AI
打开浏览器访问：**http://localhost:8000/docs**

---

## 📊 状态检查

启动后，检查以下三个地址：

| URL | 用途 | 预期响应 |
|-----|------|---------|
| http://localhost:8000/health | 健康检查 | `{"status": "ok"}` |
| http://localhost:8000/profile | AI 信息 | AI 状态和配置 |
| http://localhost:8000/docs | **API 测试** | Swagger UI 界面 |

## 🤔 没有 API Key？

### 快速获取：
1. 访问 https://platform.openai.com/api-keys
2. 创建新 API Key
3. 复制 `sk-...` 开头的密钥
4. 粘贴到上面的命令中

### 或者先测试环境：
```bash
python start_ai_child.py --check
```

## 🚨 常见问题

### Q1：端口 8000 已被占用
**A**：编辑 `server/main.py`，改变启动端口
```bash
# 或使用其他端口启动
uvicorn main:app --port 9000
```

### Q2：找不到 start_ai_child.py
**A**：确保在项目根目录运行
```bash
cd /Volumes/mac第二磁盘/ai-child
python start_ai_child.py --server
```

### Q3：API Key 无效
**A**：检查 `.env` 文件格式
```bash
cat server/.env
# 应该显示：OPENAI_API_KEY=sk-xxxxx
```

### Q4：虚拟环境错误
**A**：重新初始化
```bash
python start_ai_child.py --check
rm -rf server/.venv
python start_ai_child.py --server
```
