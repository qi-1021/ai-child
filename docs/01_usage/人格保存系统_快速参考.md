## 💾 数据库表速查

### AIProfile (人格表)
```sql
CREATE TABLE ai_profile (
  id INTEGER PRIMARY KEY = 1,              -- 单例
  name VARCHAR(128) = NULL,                 -- AI 名字
  created_at DATETIME,
  named_at DATETIME,
  is_sleeping BOOLEAN = FALSE,             -- 睡眠状态
  last_consolidation_summary TEXT         -- 睡眠摘要
);
```

### 相关表
- `conversations` - 对话历史
- `knowledge` - 知识库
- `pending_questions` - 待答问题 (topic="__name__" 特殊)
- `tools` - 自创工具
- `sleep_events` - 睡眠事件

## 🔄 命名流程关键时刻

```
1️⃣ 服务器启动
   ↓ ensure_name_question_exists()
   ↓
2️⃣ 检查 AI 是否已命名
   ├─ YES → 无操作
   └─ NO → 创建 PendingQuestion(topic="__name__")
   ↓
3️⃣ 用户通过 API/Telegram 回答
   ↓ POST /teach/questions/{id}/answer
   ↓
4️⃣ 回答处理 (api/teach.py#78-107)
   ├─ 检测 topic == "__name__"
   ├─ 调用 extract_name_from_answer()
   │  └─ GPT-4o: "就叫小明吧" → "小明"
   ├─ 调用 set_ai_name(session, "小明")
   │  └─ 更新 AIProfile.name + named_at
   └─ 返回确认消息
   ↓
5️⃣ 名字永久保存 ✅
```

## 🌙 睡眠周期中的状态变化

```
22:00 (睡眠时刻)
├─ AIProfile.is_sleeping = True
├─ consolidate_memories() 后台任务
│  └─ 生成 last_consolidation_summary
├─ 创建 SleepEvent(type="sleep")
└─ 系统提示变为睡眠语气

07:00 (唤醒时刻)
├─ AIProfile.is_sleeping = False
├─ 创建 SleepEvent(type="wake")
└─ 系统提示恢复正常语气
```

## 🛠️ 常用操作代码片段

### 获取 AI 名字
```python
from ai.profile import get_ai_name
from models import async_session

async with async_session() as session:
    name = await get_ai_name(session)
    if name:
        print(f"AI 名字: {name}")
    else:
        print("AI 尚未命名")
```

### 查询睡眠状态
```python
from ai.profile import get_or_create_profile
from models import async_session

async with async_session() as session:
    profile = await get_or_create_profile(session)
    if profile.is_sleeping:
        print(f"AI 正在睡眠... 摘要: {profile.last_consolidation_summary}")
    else:
        print("AI 清醒中")
```

### 列出未回答问题
```python
from ai.memory import get_unanswered_questions
from models import async_session

async with async_session() as session:
    questions = await get_unanswered_questions(session)
    for q in questions:
        print(f"问题 {q.id}({q.topic}): {q.question}")
```

## 🚨 常见陷阱

| 陷阱 | 后果 | 解决方案 |
|-----|------|--------|
| 多次调用 `get_or_create_profile()` | ❌ 多个 Profile 实例 | ✅ 已处理（id=1 幂等） |
| 在 AI 已命名后再调用 `ensure_name_question_exists()` | ❌ 重复创建问题 | ✅ 已检查（if name is not None) |
| 忘记更新 `is_sleeping` 时刻 | ❌ 状态不一致 | ✅ sleep_scheduler 自动管理 |
| 导出备份不含敏感信息加密 | ❌ 隐私泄露 | 🔄 建议实现加密选项 |

## 📈 字符限制和约束

| 字段 | 限制 | 备注 |
|-----|------|------|
| `name` | ≤ 64 字符 | 由 `extract_name_from_answer()` 强制 |
| `topic` | ≤ 256 字符 | 数据库定义 |
| `content` | TEXT (无限) | 支持长论述 |
| 对话上下文 | 默认 20 条 | `memory_context_turns` 配置 |

## ✨ 最佳实践

1. **始终通过** `get_or_create_profile()` 访问人格
   ```python
   # ✅ 好
   profile = await get_or_create_profile(session)
   
   # ❌ 错误
   profile = AIProfile(id=1)  # 可能不同步
   ```

2. **使用** `set_ai_name()` 保存名字
   ```python
   # ✅ 好
   await set_ai_name(session, name)
   
   # ❌ 错误
   profile.name = name
   await session.commit()  # 不更新 named_at
   ```

3. **响应命名问题时检查 topic**
   ```python
   # ✅ 好
   if question.topic == NAME_QUESTION_TOPIC:
       # 特殊处理
   
   # ❌ 错误
   if "name" in question.question.lower():
       # 不可靠
   ```

4. **导出备份前做验证**
   ```python
   # ✅ 好
   if profile.name is None:
       warn("导出的是未命名的 AI 实例")
   ```
