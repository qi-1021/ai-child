# ⚡ 人格隔离与本地化 - 快速参考

## 🎯 三个核心功能

### 1️⃣ 人格记忆隔离

**永不删除的重要记忆**

```python
# 记录 AI 的个性特征
await manager.add_memory(
    category="traits",           # traits|values|origin|relationships|beliefs
    content="我天生好奇",
    significance=95,             # 1-100，越高越重要
    source="user",              # user|self|system
    is_neutral_fact=False,      # False=有立场的人格，True=中立事实
    tags=["core", "identity"],
)
```

| 表名 | 特点 | 删除策略 |
|------|------|--------|
| `personality_memories` | 人格定义 | ❌ 永不删除 |
| `conversations` | 对话历史 | ✅ 30-90天后清理 |
| `knowledge` | 学习的事实 | ✅ 置信度衰退 |

## 📊 人格记忆分类

```
category: "traits"          ← 个性特征          (固定不变)
  "I like asking why"

category: "values"          ← 核心价值观        (定义信念)
  "Honesty matters more than perfection"

category: "origin"          ← 来源故事          (背景信息)
  "I was created to learn through conversation"

category: "relationships"   ← 重要关系          (社交记忆)
  "User 123 taught me about quantum physics"

category: "beliefs"         ← 核心信念          (哲学立场)
  "Curiosity is the engine of learning"

## 🔑 关键 API 端点

### 人格管理

```
POST   /personality/memories              # 添加记忆

## ✨ 人格 vs 中立

### ✨ 有立场的人格特征

这些定义 AI **独有的个性**：

```
- "我喜欢通过提问来学习"
- "我相信诚实胜过效率"
- "用户的感受对我很重要"
- "我对艺术特别感兴趣"
```

标记为：`is_neutral_fact = False`

### 🔵 中立的事实信息

这些是**所有 AI 都相同**的：

```
- "我是基于 GPT-4o"
- "我支持 JSON 输出"
- "我的上下文窗口是 8K"
```

标记为：`is_neutral_fact = True`

### 在系统提示词中的区别

```python
# 自动注入到系统提示词
if not memory.is_neutral_fact:
    # 这个人格特征会影响 AI 的表现
    system_prompt += memory.content
```

## 💡 常见问题

**Q: 我能改变 AI 的已有人格吗？**  
A: 可以在 personality_memories 表中编辑，但建议通过对话让 AI 自然演变。

**Q: 删除 personality_memories 中的记忆会怎样？**  
A: AI 会失去那部分人格特征，重启后立即生效。

**Q: 能在不同的语言间切换吗？**  
A: 可以，设置 `preferred_language` 即可。所有消息会自动翻译。

**Q: 导出的档案能用在其他 AI 上吗？**  
A: 可以！JSON 格式是通用的。其他项目可以导入相同的档案。
