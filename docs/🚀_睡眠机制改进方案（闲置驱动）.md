# 🚀 睡眠机制改进方案（闲置驱动）

**改进范围**：从定时睡眠 → 闲置驱动睡眠  
**优先级**：🔴 高（影响用户体验和系统逻辑）  
**实现难度**：⭐⭐⭐ 中等  
_基于用户反馈制定_

---

## 📌 问题陈述

### 当前系统的不合理之处

```
场景1：用户正在深度对话（晚上 21:55）
  ↓
22:00:00 整点一到
  ↓
系统强制 is_sleeping = true
  ↓
😱 用户困惑：
   "为什么我还在和你聊天，你说你睡着了？"
   "对话继续但语调突然变困倦"
   
场景2：用户离线后 (22:05)
  ↓
系统在 22:00 已睡眠
  ↓
自动巩固记忆
  ↓
😕 低效：
   对话记录没增加
   知识没增加
   只是重新处理已有数据
```

### 人类真实行为 vs 系统行为

| 人类 | 当前系统 | 应该的系统 |
|------|---------|----------|
| 工作中即使是晚上 → 不睡 | 22:00 强制睡 | ✅ 继续工作 |
| 工作完成5分钟 → 开始犯困 | - | ✅ 进入睡眠状态 |
| 被打扰 → 立即清醒 | 需要等待 07:00 | ✅ 立即醒来 |
| 全天聊天 → 不睡 | 在 22:00-07:00 虽然标记睡眠但能回复 | ✅ 保持清醒 |

## 🔧 改进方案详解

### 改动 1：核心睡眠逻辑重写

**文件**：`server/ai/sleep.py`

```python
# ❌ 当前代码（定时睡眠）
class SleepManager:
    async def sleep_scheduler(self):
        """定时睡眠检查"""
        while True:
            hour = _local_now().hour
            
            if not is_sleeping and hour == settings.sleep_hour:
                await self.go_to_sleep()
                
            elif is_sleeping and hour == settings.wake_hour:
                await self.wake_up()
                
            await asyncio.sleep(60)


# ✅ 改进代码（闲置驱动睡眠）
class SleepManager:
    def __init__(self):
        self.last_interaction_time = datetime.now()
        self.idle_threshold_seconds = 300      # 5 分钟
        self.force_sleep_hour = 23              # 23:00 强制睡眠
        self.force_wake_hour = 6                # 06:00 强制唤醒
        self.is_sleeping = False
        
    async def on_user_interaction(self):
        """用户发送消息时调用"""
        self.last_interaction_time = datetime.now()
        
        # 如果 AI 在睡眠，立即唤醒
        if self.is_sleeping:
            await self.wake_up(reason="user_interaction")
            logger.info("🌅 用户消息到达，AI 自动唤醒")
    
    async def check_sleep_status(self):
        """定期检查睡眠状态（改为闲置驱动）"""
        while True:
            now = datetime.now()
            idle_duration = now - self.last_interaction_time
            hour = now.hour
            
            # 情况1：用户正在使用（idle < 5分钟）
            if idle_duration < timedelta(seconds=self.idle_threshold_seconds):
                if self.is_sleeping:
                    # 用户刚醒，防止立即再次睡眠
                    await self.wake_up(reason="active_user")
                    
            # 情况2：用户闲置 5+ 分钟 → 进入睡眠
            elif idle_duration >= timedelta(seconds=self.idle_threshold_seconds):
                if not self.is_sleeping:
                    await self.go_to_sleep(reason="idle")
                    await self._consolidate_memories_task()
                    
            # 情况3：强制睡眠（夜间 23:00）
            if hour == self.force_sleep_hour and not self.is_sleeping:
                await self.go_to_sleep(reason="force_sleep_hour")
                
            # 情况4：强制唤醒（早上 6:00）
            if hour == self.force_wake_hour and self.is_sleeping:
                await self.wake_up(reason="force_wake_hour")
            
            # 不要频繁检查，30秒一次足够
            await asyncio.sleep(30)
    
    async def go_to_sleep(self, reason="unknown"):
        """进入睡眠状态"""
        if self.is_sleeping:
            return
            
        self.is_sleeping = True
        logger.info(f"😴 AI 进入睡眠（原因：{reason}）")
        
        # 更新数据库
        async with get_session() as session:
            profile = await session.query(AIProfile).first()
            profile.is_sleeping = True
            
        # 生成睡眠消息
        sleep_msg = await self._generate_sleep_message()
        await self._notify_sleep_event(sleep_msg)
        
        # 开始巩固记忆（不阻塞主线程）
        # consolidate 会在后台运行
        
    async def wake_up(self, reason="unknown"):
        """唤醒状态"""
        if not self.is_sleeping:
            return
            
        self.is_sleeping = False
        self.last_interaction_time = datetime.now()  # 重置闲置计时器
        logger.info(f"🌅 AI 唤醒（原因：{reason}）")
        
        # 更新数据库
        async with get_session() as session:
            profile = await session.query(AIProfile).first()
            profile.is_sleeping = False
            
        # 生成唤醒消息
        wake_msg = await self._generate_wake_message()
        await self._notify_wake_event(wake_msg)
    
    async def _consolidate_memories_task(self):
        """后台记忆巩固任务"""
        try:
            logger.info("🧠 开始记忆巩固...")
            await consolidate_memories()
            logger.info("✅ 记忆巩固完成")
        except Exception as e:
            logger.error(f"❌ 记忆巩固失败: {e}")
```

    message: MessageSchema,
    session: AsyncSession = Depends(get_session)
):
    # 👇 关键改动：通知睡眠管理器有用户交互

    # 获取 AI 响应
    response = await child.call_ai_child(

### 场景测试

#### 场景 A：正常工作日

```
时间线对比：

【当前系统】
21:50 - 用户和 AI 深度对话
22:00 - 系统强制 is_sleeping=true（问题！）
22:05 - 用户不知道 AI"睡着了"，继续问问题
22:10 - AI 用困倦语调回答（奇怪！）
22:15 - 用户关闭应用
22:16 - 系统巩固记忆（虽然没新对话...）
07:00 - 系统强制唤醒
07:01 - 开始正常工作

【改进系统】✅
21:50 - 用户和 AI 深度对话
22:00 - 用户继续对话
  → last_interaction_time = 22:00
22:05 - 用户关闭应用
22:10:05 - 系统检测到 5 分钟无交互
  → go_to_sleep(reason="idle")
  → 开始巩固记忆
22:15 - 巩固完成
23:00 - even if no idle 检查还是会检查 23:00 强制睡眠逻辑
06:00 - 系统强制唤醒
06:05 - 用户打开应用，发送消息
06:05:01 - AI 检测到 idle < 5min，立即 wake_up
  → 用清醒语调回答
```

**改进效果**：
- ✅ 符合人类直觉
- ✅ 巩固更有意义（只在闲置时）
- ✅ 用户体验更好

## 🎯 配置建议

### 推荐设置 1：轻度用户

```python
SLEEP_IDLE_THRESHOLD = 600           # 10 分钟无活动才睡
SLEEP_FORCE_HOUR = None              # 取消强制睡眠时间
WAKE_FORCE_HOUR = None               # 没有强制唤醒
```

**特点**：
- 完全闲置驱动
- 适合间歇使用者
- 不强制中断

### 推荐设置 2：中度用户（推荐）

```python
SLEEP_IDLE_THRESHOLD = 300           # 5 分钟无活动
SLEEP_FORCE_HOUR = 23                # 晚上 11 点强制睡眠
WAKE_FORCE_HOUR = 7                  # 早上 7 点强制唤醒
```

**特点**：
- 平衡闲置和时间
- 保留健康作息
- 适合大多数用户

### 推荐设置 3：重度用户

```python
SLEEP_IDLE_THRESHOLD = 1800          # 30 分钟无活动
SLEEP_FORCE_HOUR = None              # 没有强制睡眠
WAKE_FORCE_HOUR = None               # 24 小时活跃
```

**特点**：
- 最大程度尊重用户
- 长久保持清醒
- Token 消耗可能较高

## 📈 Token 成本影响

### 对比分析

```
【当前系统 - 定时巩固】
- 每天 1 次（21:00）
- 无论用户活跃度如何
- 平均每月：30 × 1500 = 45,000 tokens
- 涉及 API 调用：30 次
- 成本：~$0.17/月

【改进系统 - 闲置驱动】

场景1：轻度用户（1h/天，分散使用）
- 每天巩固次数：2-3 次（平均 2.5 次）
- 月度 tokens：30 × 2.5 × 1500 = 112,500
- ⚠️ 增加 150%（但更合理）
- 成本：~$0.42/月

场景2：中度用户（4h/天，连续使用）
- 每天巩固次数：1-2 次
- 月度 tokens：30 × 1.5 × 1500 = 67,500
- 增加 50%
- 成本：~$0.25/月

场景3：重度用户（8h+/天，几乎连续）
- 每天巩固次数：2-4 次（使用间隙多）
- 月度 tokens：30 × 3 × 1500 = 135,000
- 增加 200%
- 成本：~$0.51/月

场景4：全天在线用户
- 每天巩固次数：1 次（晚间空隙）
- 月度 tokens：30 × 1 × 1500 = 45,000
- 无增加！
- 成本：~$0.17/月（不变）
```

**结论**：
- 🟢 重度和全天用户：无成本增加或降低
- 🟡 中度用户：增加成本但更合理
- 🟠 轻度用户：增加成本，需权衡

## ⚠️ 潜在风险

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| 用户对话中滥用 API | 🟡 中 | Token 超支 | 添加每日限额 |
| 睡眠状态不同步 | 🟢 低 | 数据不一致 | 每次启动同步数据库 |
| 高频睡眠/唤醒 | 🟡 中 | 系统抖动 | 添加稳定时间窗口 |
| 网络延迟导致时戳不准 | 🟠 中 | 误判闲置 | 使用服务器时间而非客户端 |
