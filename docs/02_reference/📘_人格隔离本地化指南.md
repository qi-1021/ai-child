## 1️⃣ 人格记忆隔离系统

### 核心概念

在之前的系统中，所有记忆平等对待。现在区分为两类：

```
┌─────────────────────────────────────────┐
│         AI Child 记忆体系                 │
├─────────────────────────────────────────┤
│                                          │
│  ✨ 人格记忆 (Personality Memories)     │
│  ├─ 永不删除，是 AI 身份的基石          │
│  ├─ 分类: traits, values, origin,      │
│  │        relationships, beliefs       │
│  ├─ 例: "我喜欢提问" "我的名字是..."   │
│  └─ 自动注入系统提示词                 │
│                                          │
│  📚 普通记忆 (Regular Memories)         │
│  ├─ 可能被清理 (30-90天规则)           │
│  ├─ 表: conversations, knowledge      │
│  ├─ 例: 用户说的日常对话、学到的事实   │
│  └─ 置信度加权系统                     │
│                                          │
└─────────────────────────────────────────┘
```

### 数据库结构

#### 新表: `personality_memories`

```sql
CREATE TABLE personality_memories (
    id INTEGER PRIMARY KEY,
    category VARCHAR(64)                  -- traits, values, origin, relationships, beliefs
    content TEXT NOT NULL,                -- 记忆内容
    significance INTEGER (1-100),         -- 重要程度 (100=基础, 1=次要)
    source VARCHAR(32),                   -- "user" | "self" | "system"
    is_neutral_fact BOOLEAN,              -- False = 有立场的人格观, True = 中立事实
    created_at DATETIME,
    last_reviewed_at DATETIME,
    tags JSON,                            -- ["important", "core", "recent"]
    context TEXT                          -- 额外背景
);
```

#### 扩展: `ai_profile` 表

新增字段：
```python
preferred_language: str = "en-US"         # 语言偏好
personality_traits: JSON = {}             # 结构化个性特征
```

### 使用示例

#### 示例 1：记录 AI 的核心特质

```python

async def teach_personality():

    # 记录 AI 的好奇心

        category="traits",

    # 记录 AI 的价值观

        category="values",

    )
```

#### 示例 2：获取用于系统提示词的人格背景

```python
# 在生成系统提示词时自动注入

    # 注入人格记忆

    # 结果包含人格背景，确保 AI 行为一致
```

```python

async def handle_user_message(text, session):
```

| `en-US` | English (US) | 100% |
| `zh-CN` | 简体中文 | 100% |

### 翻译用法

#### 方法 1：全局 i18n 实例

```python
from server.i18n import get_i18n

# 创建或获取 i18n 实例
i18n = get_i18n("zh-CN")

# 翻译消息
msg = i18n.t("sleep.message.wake")
# 输出: "早上好！我一直在思考我们的对话。我有一些新问题想问你！"

# 带变量插值
greeting = i18n.t("name_accepted", ai_name="小明")
# 输出: "很高兴认识你！你可以叫我 小明。"
```

#### 方法 2：快速翻译

```python
from server.i18n import t

# 一次性翻译
msg = t("greeting", language="zh-CN")
msg = t("error.api_error", language="en-US")
```

#### 方法 3：系统提示词

```python
i18n = get_i18n("zh-CN")
system_prompt = i18n.system_prompt(ai_name="小明", is_sleeping=False)
```

### 添加新翻译

编辑 `/server/i18n/messages.py`：

```python
MESSAGES = {

```

然后使用：

## 3️⃣ 人格档案导出/导入

### 导出人格档案

用于 GitHub 备份：

```bash
# 导出为 JSON
python export_personality.py
# 输出: personality_profile.json

# 导出为 Markdown（易读）
python export_personality.py --format markdown
# 输出: personality_profile.md

# 导出为 YAML
python export_personality.py --format yaml
# 输出: personality_profile.yaml

# 带时间戳的备份
python export_personality.py --backup-dir ./backups
# 输出: backups/personality_profile_20260321_102530.json
```

## 5️⃣ 中立 vs 有立场

### 关键区别

**中立的AI助手信息** (`is_neutral_fact=True`):
- "我是由 OpenAI 创建的"
- "我支持 Python 3.10+"
- "温度参数范围是 0-2"

这些信息对所有 AI 都相同，不是个性特征。

**有立场的人格特征** (`is_neutral_fact=False`):
- "我喜欢帮助人解决难题"
- "我认为诚实比效率更重要"
- "我对艺术特别感兴趣"

这些是这个特定 AI 的独特观点。

### 在系统提示词中的体现

```python
# 只有 is_neutral_fact=False 的记忆会被注入系统提示词
async def highlight_personality_in_system_prompt(prompt, session):
    memories = await get_core_memories()
    
    for mem in memories:
        if not mem.is_neutral_fact:  # ← 关键：过滤个性特征
            # 注入人格观
            context += f"- {mem.content}"
    
    return prompt_with_personality
```

## 💬 FAQ

**Q: 人格记忆和普通记忆的区别是什么？**  
A: 人格记忆（如"我喜欢问问题"）定义 AI 的身份，永不删除。普通记忆（如对话历史）可以根据时间清理。

**Q: 能改变 AI 的语言偏好吗？**  
A: 可以，通过 `profile.preferred_language` 字段或 `i18n.set_language()` API。

**Q: 导出的档案可以在其他项目中使用吗？**  
A: 可以！JSON 格式是标准的，其他 AI 服务可以导入同样的档案。

**Q: 个人观点（有立场）会影响功能吗？**  
A: 不会，它只在系统提示词中显示，帮助 AI 表现出一致的人格。
