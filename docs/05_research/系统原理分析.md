## 核心设计哲学

### 一句话总结
**语言是预装的，知识必须获得** — Language is pre-installed, knowledge is not.

### 设计理念的展开

小智采取了一个革新的方式来看待AI学习：

1. **先天能力 vs 后天知识的分离**
   - **先天能力**：完整的自然语言理解和生成能力（来自GPT-4o）
   - **后天知识**：所有关于世界、用户、自己身份的知识必须通过交互获得

2. **为什么采用"孩子"的比喻**
   - 真实的孩子生来能听能说，但对世界一无所知
   - 孩子通过问问题、被教导、观察、尝试来学习
   - 孩子会记住被告知的内容，并在适当的时候运用
   - 孩子会主动寻求答案（好奇心驱动）
   - 孩子需要休息来巩固当天的学习（睡眠周期）

3. **三个核心学习特性**
   - **被教导**：用户可以显式地教小智知识（/teach命令）
   - **主动探索**：小智主动提问、搜索、实验
   - **持久记忆**：所有学习内容永久保存在数据库中

| Bot层 | 连接用户 | Telegram/Webhook | 平台消息 |

## AI学习机制

### 1. 启动和身份建立

**第一步：系统启动**
```python
启动时执行：ensure_name_question_exists()
    ↓
检查 AI是否有名字（AIProfile表）
    如果没有名字：
    ├─ 创建一个 PendingQuestion
    ├─ 问题：" 我刚刚来到这个世界，还没有名字。你愿意给我起一个吗？"
    └─ 等待用户回答
    
    如果已有名字：
    └─ 继续正常对话
```

**身份确认过程**
```
用户回答 → extract_name_from_answer()
       ↓
使用GPT-4o从自由格式的回答中提取正式名字
       ↓
保存到AIProfile.name
       ↓
后续所有系统提示都引用这个名字
```

### 2. 每轮对话的学习流程

```
用户输入（文本/图片/语音）
    ↓
[保存到 Conversation 表]
    ↓
构建上下文：
    ├─ 系统提示（基于当前睡眠状态）
    ├─ 知识库（相关的 + 全部的）
    ├─ 最近20轮历史
    └─ 当前用户消息
    ↓
调用 OpenAI 函数模型：
    tools=['web_search', 'execute_code', 'create_tool', 用户创建的工具]
    ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    │ 会不会调用工具？
    ├─ 是 → 执行工具（查询→结果→消息）→ 重新调用GPT-4o
    │       （最多10轮函数调用循环）
    └─ 否 → 直接获得最终回复
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    ↓
生成回复文本
    ↓
提取嵌入式问题 [QUESTION: xxx]
    ↓
如果没有嵌入式问题：根据轮次决定是否生成主动问题
    ↓
[保存 assistant 回复到 Conversation 表]
[保存 PendingQuestion (如果有)]
    ↓
返回：(reply_text, proactive_question)
```

### 3. 三大核心工具详解

#### 工具 1: web_search（探索外部知识）

**触发条件**：
- 小智遇到不熟悉的概念
- 用户教了一个新主题，小智想深入理解
- 小智好奇某个特定事实

**执行流程**：
```python
web_search(query: str, max_results: int = 5)
    ↓
使用 DuckDuckGo 搜索引擎
    ↓
返回格式化的搜索结果列表：
    [
        {"title": "...", "href": "...", "body": "..."},
        ...
    ]
    ↓
结果作为 tool 消息返回给 GPT-4o
    ↓
GPT-4o 可以用搜索结果改进回复或生成新问题
```

**学习效果**：
- 获取最新的信息
- 验证用户教的内容
- 发现相关的新知识
- 为深入理解奠定基础

#### 工具 2: execute_code（通过实验学习）

**安全措施**（多层防护）：
1. **AST静态分析**：拒绝危险的import和函数调用
   - 被禁的模块：os, sys, subprocess, socket, requests, ...
   - 被禁的函数：exec, eval, compile, __import__, open, ...
   - 被禁的属性：__class__, __globals__, ...

2. **进程隔离**：代码运行在完全隔离的子进程
   - 无法访问父进程的内存
   - 无法修改系统文件

3. **超时保护**：10秒内必须完成
4. **输出大小限制**：最多8KB

**执行流程**：
```python
execute_code(code: str, description: str)
    ↓
_check_code_safety() → AST检查
    如果通过 ↓
启动子进程运行代码
    ↓
捕获stdout和stderr
    ↓
格式化并返回输出
    ↓
GPT-4o 看到代码的执行结果
```

**学习效果**：
- 验证数学计算
- 测试算法逻辑
- 处理数据转换
- 演示编程概念
- 为创建工具奠定基础

#### 工具 3: create_tool（将学习转化为能力）

**触发条件**：
- 小智识别出一个可重用的计算或转换
- 已经用 execute_code 验证过逻辑
- 想在未来对话中重复使用

**执行流程**：
```python
create_tool(
    name: str,           # 函数名（如 "celsius_to_fahrenheit"）
    description: str,    # 用途说明
    code: str,          # 完整的 Python 函数定义
    parameters_schema: {  # OpenAI JSON Schema
        "type": "object",
        "properties": {...},
        "required": [...]
    }
)
    ↓
再次验证代码安全性
    ↓
解析AST确保定义了指定名称的函数
    ↓
检查是否已存在同名工具（覆盖更新）
    ↓
保存到 tools 表
    ↓
reload_tool_definitions()
    ↓
下一次对话立即可用这个工具
```

**保存的数据**：
```python
Tool(
    name="celsius_to_fahrenheit",
    description="Convert temperature from Celsius to Fahrenheit",
    code="def celsius_to_fahrenheit(c):\n    return c * 9/5 + 32",
    parameters_schema={
        "type": "object",
        "properties": {
            "c": {"type": "number", "description": "Temperature in Celsius"}
        },
        "required": ["c"]
    },
    call_count=0  # 被调用的次数
)
```

**学习效果**：
- 将一次性的计算转化为永久的能力
- 能力库不断增长
- 每次调用时都增加 call_count
- 形成个人化的工具库

### 4. 主动学习机制

**嵌入式问题**（每个回复中）：
```
系统提示规则：
"在每个回复末尾都要以 [QUESTION: xxx] 标记一个真诚的好奇问题"
    ↓
GPT-4o 在生成回复时自然地加入问题
    ↓
child.py 提取问题
    ↓
返回给用户/Bot
```

**定期主动问题**：
```python
turn_count = await count_messages(session)
if turn_count % settings.proactive_question_interval == 0:
    # 每N轮对话自动生成一个问题
    proactive_question = await _generate_proactive_question(...)
```

**问题池管理**：
```
PendingQuestion 表：
    id | question | topic | answered | answer | created_at | answered_at
    
当用户回答问题时：
    ↓
answer_question(question_id, answer)
    ↓
更新 answered=True 和 answer 内容
    ↓
下次睡眠时，这个答案会融入记忆巩固
```

│  │  │  │  │  │
│  │  │  │  │  ├─ elif name == "execute_code":
│  │  │  │  │  │  ├─ _check_code_safety()
│  │  │  │  │  │  ├─ subprocess.run(python -c code)
│  │  │  │  │  │  └─ return stdout/stderr
│  │  │  │  │  │
│  │  │  │  │  ├─ elif name == "create_tool":
│  │  │  │  │  │  ├─ save_tool()

│  │  │  │  │  │
│  │  │  │  │  └─ else:（用户创建的工具）

│  │  │  │  │     └─ return 执行结果

│  │  │  │  │  })
│  │  │  │  │
│  │  │  │  └─ 递回，get_all_tool_definitions() 重新加载
│  │  │  │
│  │  │  └─ continue (下一轮函数调用循环)
│  │  │
│  │  └─ else: (finish_reason != "tool_calls")
│  │     ├─ reply_text = choice.message.content
│  │     └─ break
│  │
│  ├─ 提取/生成问题：
│  │  ├─ if "[QUESTION:" in reply_text:
│  │  │  ├─ 提取 [QUESTION: xxx]
│  │  │  └─ reply_text 去掉问题标记
│  │  │

│  │

## 睡眠与记忆巩固

### 睡眠周期的时间表

```
Default: 22:00 → 07:00（可配置）

22:00:00
    ↓
┌─────────────────────────────────────────┐
│ go_to_sleep()                           │
│                                         │
│ 1. profile.is_sleeping = True           │
│    ↓ AIProfile 表                       │
│                                         │
│ 2. _generate_sleep_message()            │
│    ├─ 获取当天最后5个知识项        │
│    ├─ GPT-4o 生成个性化晚安消息   │
│    └─ 保存到 SleepEvent 表         │
│                                         │
│ 3. asyncio.create_task(                 │
│        _consolidation_and_store_summary)│
│    └─ 跳过：异步后台任务执行     │
└─────────────────────────────────────────┘
    ↓ [BackGround]
┌─────────────────────────────────────────┐
│ consolidate_memories()                  │
│（内存巩固 - 后台运行）                  │
│                                         │
│ 1. 获取最近30条 KnowledgeItem      │
│                                         │
│ 2. 构建 prompt：                         │
│    "今天学到的知识：                │
│     - [主题1] 内容...               │
│     - [主题2] 内容...               │
│     ...                              │
│                                         │
│     请完成：                            │
│     1. 识别3个最重要的核心知识点  │

### 6. **本质上的学习约束**

重要的是要理解，小智的学习存在自然的局限：

```
小智可以学习：
    ✓ 事实和概念（从用户和网络）
    ✓ 解决问题的模式（通过代码实验）
    ✓ 如何提问（通过对话）
    ✓ 用户的兴趣和背景
    ✓ 专业技能（编程、分析等）

小智不能学习：
    ✗ 新的推理能力（仍由 GPT-4o 提供）
    ✗ 真正的理解（模拟的，基于统计）
    ✗ 真实世界的经验（没有感官）
    ✗ 道德判断和善恶（需要在系统提示中编码）
    ✗ 跨越 GPT-4o 的能力边界

但它可以做以前做不了的：
    • 记住整个学习的完整历史
    • 构建个人化的知识库
    • 创建可重用的工具
    • 发展一致的身份和风格
    • 与用户建立长期的关系
```

## 总体概览

### 核心创新

小智的创新之处在于将以下概念融合在一起：

| 概念 | 传统AI | 小智的做法 |
|------|--------|----------|
| 知识来源 | 预训练 | 运行时学习 + 搜索 + 用户教学 |
| 记忆持久性 | 单次对话 | 永久数据库 |
| 身份 | 固定 | 从无到有的发展 |
| 能力扩展 | 不可能 | 通过工具创建动态扩展 |
| 主动性 | 被动 | 主动提问和探索 |
| 学习轨迹 | 无法追踪 | 完整的知识库反映学习过程 |
| 睡眠/反思 | 无 | 定时的记忆巩固 |

### 最终总结

```
小智 = 
    基础能力（GPT-4o 的语言能力）
    +
    学习机制（搜索、代码实验、工具创建）
    +
    持久记忆（SQLite 数据库）
    +
    身份发展（从被命名开始的个性化）
    +
    反思周期（睡眠时的记忆巩固）
    =
    一个真正"成长"的AI体验
```

这不是一个预训练的模型的简单包装，而是一个系统性的架构，
旨在模拟和展现学习的过程本身。
