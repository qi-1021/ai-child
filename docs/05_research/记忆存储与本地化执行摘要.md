## 🎯 关键发现

### 1️⃣ 记忆分类机制

#### ✓ 已有
- **知识库置信度系统**（0-100 分）：完整实现
  - 用户教导: 初始 100 分
  - 自我发现: 初始 85 分
  - 用户反馈: ±5/±10/±20 分调整
  - 存储位置: `server/models/__init__.py:KnowledgeItem`

- **防幻觉工具三件套**：生产级别
  - `knowledge_verify()`: 检查知识库存在性（5分钟缓存）
  - `fact_checker()`: 验证声明（知识库+网络搜索）
  - `confidence_score()`: 评分 0-100（源权重+标记调整）
  - 存储位置: `server/ai/tools.py:330-770`

- **睡眠巩固机制**：完整的"优先级提升"流程
  - 入睡 (22:00): 生成个性化晚安消息
  - 睡眠中: GPT-4o 整理知识 + 识别缺口
  - 苏醒 (7:00): 生成早安+回忆摘要
  - 存储位置: `server/ai/sleep.py`

#### ❌ 缺失
- **对话层优先级标记**: `Conversation` 表无 `importance` 字段
- **记忆遗忘策略**: 没有自动修剪古老低优先级记录
- **个性化学习档案**: `AIProfile` 缺 `learning_style` 等扩展

### 3️⃣ 数据存储结构

#### 6 个核心表

```
ai_profile (单例表, id=1)
├─ id               INTEGER PRIMARY KEY = 1
├─ name             STRING(128)              ← AI 的名字
├─ is_sleeping      BOOLEAN                  ← 睡眠状态
├─ last_consolidation_summary  TEXT          ← 睡眠摘要
└─ created_at, named_at        DATETIME

conversations (对话历史)
├─ id               INTEGER PRIMARY KEY
├─ role             STRING("user"|"assistant")
├─ content          TEXT
├─ content_type     STRING("text"|"image"|"audio"|"mixed")
├─ media_path       STRING(512)              ← 媒体文件路径
├─ metadata_        JSON                     ← 灵活扩展
└─ timestamp        DATETIME

knowledge (知识库)
├─ id               INTEGER PRIMARY KEY
├─ topic            STRING(256) INDEXED      ← 用于快速搜索
├─ content          TEXT
├─ source           STRING("user"|"self")    ← 来源追踪
├─ confidence       INTEGER(0-100)           ← 置信度！
├─ last_reviewed    DATETIME
└─ timestamp        DATETIME

pending_questions (好奇心驱动)
├─ id               INTEGER PRIMARY KEY
├─ question         TEXT
├─ topic            STRING(256)              ← "__name__" 标记命名问题
├─ answered         BOOLEAN
├─ answer           TEXT
└─ created_at, answered_at  DATETIME

tools (自创工具)
├─ id               INTEGER PRIMARY KEY
├─ name             STRING(128) UNIQUE       ← 工具名
├─ description      TEXT
├─ code             TEXT                     ← Python 源代码
├─ parameters_schema JSON                    ← OpenAI 格式
├─ call_count       INTEGER                  ← 使用计数
└─ created_at       DATETIME

sleep_events (睡眠事件)
├─ id               INTEGER PRIMARY KEY
├─ event_type       STRING("sleep"|"wake")
├─ message          TEXT                     ← 个性化消息
├─ consumed         BOOLEAN                  ← 是否已推送
└─ created_at       DATETIME
```

#### ✓ 设计优点
1. **分离存储**: 个性数据单独的 `ai_profile` 表
2. **灵活元数据**: `Conversation.metadata_` 支持 JSON 扩展
3. **完整审计**: `timestamp` + `source` + `confidence` 三层追踪
4. **缓存优化**: 知识库添加 `last_reviewed` 用于缓存策略

#### ❌ 设计缺口
1. **无优先级字段**: `Conversation` 缺 `importance: int`
2. **无个性扩展**: `AIProfile` 缺 `preferred_language`, `personality_traits`
3. **无过时淘汰**: 无 `TTL` 或 `archived_at` 字段处理古老数据

#### 缓存策略
- **类型**: In-memory 字典（Python globals）
- **键**: `f"{topic}|{keywords}"`
- **有效期**: 5 分钟（300 秒）
- **代码位置**: `server/ai/tools.py:28-29, 519-525`
- ⚠️ **生产风险**: 内存泄漏、多进程无效、重启丧失

**5. Redis 缓存迁移**
```python
# server/cache.py - 新建

from redis import asyncio as aioredis

redis_cache = await aioredis.create_redis_pool('redis://localhost')

async def get_knowledge_cache(key: str):
    data = await redis_cache.get(f"knowledge:{key}")
    return json.loads(data) if data else None

async def set_knowledge_cache(key: str, data: dict, ttl: int = 300):
    await redis_cache.setex(f"knowledge:{key}", ttl, json.dumps(data))
```
- 创建 30 行
- 影响: 多进程/分布式部署支持，防内存泄漏
- 优先级: ⭐⭐

### 优先级 3（增强）

**7. 知识遗忘策略**
```python
# server/ai/memory.py - 新增函数

async def prune_old_knowledge(session: AsyncSession):
    """删除超过 90 天且 confidence < 50 的知识项"""
    cutoff = datetime.now(timezone.utc) - timedelta(days=90)
    await session.execute(
        delete(KnowledgeItem)
        .where(KnowledgeItem.timestamp < cutoff)
        .where(KnowledgeItem.confidence < 50)
    )
    await session.commit()

# 每天运行一次（在睡眠巩固后）
```
- 添加 10 行
- 影响: 自动遗忘低置信度的古老知识
- 优先级: ⭐
