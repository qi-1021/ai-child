# 🛡️ 模型先验知识防治方案

## 问题定义

**核心矛盾**：
- **初心**：AI 从零开始学习，知识完全来自用户教导和主动探索
- **现实**：GPT-4o 内置了海量知识（Wikipedia、教科书、互联网等）
- **风险**：AI 可能混用"学到的"和"固有的"知识，破坏"从零学习"的育儿初衷

**具体场景**：
```
用户: "你知道光合作用吗？"
AI: "我听说过光合作用是…" (使用训练数据)
vs. 应该是: "我不知道。你能教我吗？" (等待用户教导)
```

```python
系统提示说明:

or found through self-directed research."
```

**评估**：⚠️ 建议级，不是强制级

### ✅ 2. 知识来源追踪（置信度系统）
```
知识源分类：
- user_taught: confidence = 100 (最可靠)
- web_researched: confidence = 40-70 (用户知道来源)
- personality_memory: 权重系统（用户设定的人格特征优于推理）
- model_training: 无显式标记 ⚠️ (我们无法区分)
```

**评估**：✅ 部分有效，关键缺陷是"model_training"无法标记

---

### ✅ 3. 防幻觉验证工具
```python
validate_knowledge_source(claim):
  - 检查声称的知识是否在用户教导的库中
  - 如果不在，标记为"需要验证"或"推理"
```

**评估**：✅ 有效，但需要主动触发

## 三层知识来源问题

| 知识类型 | 来源 | 当前状态 | 防护程度 |
|---------|------|--------|--------|
| 用户教导 | `/teach` 命令 | ✅ 追踪完整 | 100% |
| 网络搜索 | `web_search` 工具 | ✅ 置信度标记 | 95% |
| 模型先验 | GPT-4o 训练数据 | ❌ 隐式存在 | **0%** |

**模型先验知识的隐患**：
- 提示词只是"请求"，GPT-4o 可自由选择忽略
- 无法从 API 响应中判断 AI 用了哪个知识源
- GPT-4o 的知识和推理能力往往表现自然，难以区分

---

## 改进方案（多层防御）

### 方案1️⃣：增强系统提示词（轻度防护）

**策略**：在 system_prompt 中明确强调

```python
# 当前系统提示的特定部分
SYSTEM_PROMPT = """
You are an AI that learns from conversation. 

CRITICAL CONSTRAINTS:
1. 你只知道用户明确教过的内容和你自主搜索到的内容
2. 如果不确定某个知识是否已学过，必须说"我不确定这是否是我学过的"
3. 引用学到的知识时，说"你教过我…"或"我在搜索中发现…"
4. 对于训练数据中的常识（如"地球是圆的"），也要表述为"我理解地球是…"
   而不是"我知道地球是…"（这样显得像是学来的事实）
5. 当用户问你某个概念时，首先承认不知道，然后问用户或搜索

CALIBRATED EXAMPLES:
✅ 好: "我对光合作用不熟悉。你能教我吗？或者我可以搜索一下。"
❌ 不好: "光合作用是植物通过光能制造养分的过程。"
```

**实现**：修改 `server/ai/child.py` 的 `_build_system_prompt()`

**有效性**：⭐⭐⭐ (提示词友好，但 GPT-4o 可能不完全遵守)

---

### 方案2️⃣：严格的知识验证审计（中度防护）

**策略**：定期检查 AI 的所有声称的知识

```python
async def audit_knowledge_claims(conversation_history):
    """
    在每次对话后，自动检查 AI 是否使用了未授权的知识
    """
    claims = extract_factual_claims(ai_response)
    
    for claim in claims:
        # 1. 检查是否在用户教导库中
        if not claim in user_taught_knowledge:
            # 2. 检查是否在网络搜索结果中
            if not claim in web_searched_knowledge:
                # 3. 标记为"可能的模型固有知识"
                flag_as_model_knowledge(claim, confidence=0.7)

    # 生成审计报告（对用户透明）
    return audit_report
```

**适用场景**：
- 教育用途（严格追踪每一条知识的来源）
- 安全敏感场景（确保 AI 不使用预训练的强观点）

**有效性**：⭐⭐⭐⭐ (精准，但需要用户定期验证)

---

### 方案3️⃣：双模型对比验证（高度防护）

**策略**：用两个独立模型判断知识来源

```
User asks: "What is photosynthesis?"

Model A (GPT-4o full): "Photosynthesis is..."
Model B (GPT-4o + context-only): 
  在仅有 system_prompt + conversation 的上下文下回答

对比: 
- 如果 A 和 B 都能回答同样内容 → 可能是模型固有知识 ⚠️
- 如果只有 A 能详细回答，B 说"不知道" → A 用了先验知识 ⚠️  
- 如果都说"不知道" → 可放心 ✅
```

**实现成本**：💰💰💰 (每次查询调用两次 API，成本翻倍)

**有效性**：⭐⭐⭐⭐⭐ (最精准，但最昂贵)

---

### 方案4️⃣：知识来源标签体系（推荐方案）

**策略**：在 AI 回答中显式标记知识来源

```python
class KnowledgeSource(Enum):
    USER_TAUGHT = "🧑‍🏫 用户教导"  # 100% 可信
    WEB_SEARCHED = "🔍 主动搜索"  # ~70% 可信
    LEARNED_INFERENCE = "🧠 推理推断" # ~50% 可信
    MODEL_PRIORS = "❓ 模型固有知识" # 待验证

# 响应格式
response = """
光合作用是植物利用光能合成有机物的过程。
[知识来源: 🔍 主动搜索 | 置信度: 65%]
"""
```

**优势**：
- ✅ 透明化：用户清楚知识从哪里来
- ✅ 可审计：长期跟踪，发现异常
- ✅ 成本低：仅在输出阶段添加标签

**实现**：
1. 修改 `AIChild.answer()` 的响应格式
2. 在 `PersonalityMemory` 中添加 source 字段
3. 生成知识溯源报告

## 推荐实施方案

### 立即实施（强度：中 + 透明度高）

**Phase 1**：增强系统提示 + 知识来源标签
```python
# 修改 child.py
async def answer(self, ...):
    response = await self._call_gpt4o(...)
    
    # 新增：提取知识来源
    source = await self._infer_knowledge_source(response)
    
    # 新增：标记来源
    tagged_response = f"{response}\n[来源: {source}]"
    
    return tagged_response
```

**成本**：~0 (仅需代码改动)  
**有效性**：⭐⭐⭐⭐ (透明 + 可追踪)

### 中期实施（强度：高 + 自动审计）

**Phase 2**：知识验证审计（后台运行）
```python
# 在 sleep/idle 时运行
async def verify_knowledge_periodically():
    """异步审计 AI 最近的知识声称"""
    recent_responses = get_last_n_responses(50)
    
    for response in recent_responses:
        claims = extract_claims(response)
        audit_result = await audit_knowledge_claims(claims)
        
        if audit_result.suspicious_claims:
            flag_for_user_review(audit_result)
```

**成本**：~低 (后台运行，不影响实时对话)  
**有效性**：⭐⭐⭐⭐⭐

## 立法建议

建议在系统设计文档中明确声明：

> **知识来源声明**  
> 本系统的 AI 学习基于三个来源：
> 1. 用户教导（最可靠）
> 2. 主动网络搜索（中等可靠）  
> 3. **内置模型知识**（用户需要自行识别）
>
> 我们提供知识来源标签帮助用户追踪，
> 但无法完全隔离 GPT-4o 的预训练知识。
> 这是当前 LLM 技术的固有限制。
