# 🚀 记忆知识注入机制 - 快速参考

## 📌 3秒钟理解

```
【现状】每次 chat() 都注入"所有知识" → 导致大量 token 浪费
【问题】搜索结果被覆盖 → 相关性筛选形同虚设
【成本】单次调用 2500-4500 tokens（平均 3200）
【优化】Top-K 筛选可节省 50-80% 知识 token
```

## 💰 Token 成本分解

```
单次 chat() 调用:

系统提示词 (System Prompt)
├─ 身份 + 行为指南 + 防幻觉规则
└─ 💾 500-800 tokens

知识块 (Knowledge Block) ⭐ 最大成本源
├─ N条知识 × 20-50 tokens/条
└─ 💾 N × 20-50 tokens
   例: 20条 = 400-1000 tokens

对话历史 (Conversation History)
├─ 20轮对话 × ~40 tokens/轮
└─ 💾 1600-2500 tokens

当前输入 + 工具定义
└─ 💾 200-300 tokens

═══════════════════════════════════
📊 总计: ~3000-4500 tokens
📊 平均: ~3200 tokens
💰 成本: ~$0.016 / 对话 (GPT-4o)
```

**成本增长公式**: `Cost ≈ 800 + (知识数 × 30) + 1800`

## ✅ 快速修复方案

### 修复 #1: 移除知识全注 (10 分钟)

```python
# 当前 (child.py:184-188)
all_knowledge = await get_all_knowledge(session)
relevant = await search_knowledge(session, user_text[:64])
seen_ids = {k.id for k in relevant}
for k in all_knowledge:
    if k.id not in seen_ids:
        relevant.append(k)

# 改为
all_knowledge = await get_all_knowledge(session)
relevant = await search_knowledge(session, user_text[:64])

# 只保留排名前10 + 所有高置信度背景知识
high_confidence = [k for k in relevant 
                   if k.confidence >= 80][:10]
background = [k for k in all_knowledge 
              if k.confidence == 100 and k.id not in {k.id for k in high_confidence}][:5]
relevant = high_confidence + background
```

**预期节省**: `-60% 知识 token` (从 600 → 240)

### 修复 #3: 启用置信度筛选 (5 分钟)

```python
# 在 _build_context() 中修改
if knowledge_items:
    # 过滤低置信度知识
    high_quality = [k for k in knowledge_items if k.confidence >= 70]
    if not high_quality:
        high_quality = knowledge_items
    
    kb_text = "\n".join(
        f"- [{item.topic}] {item.content}" for item in high_quality

```

**预期效果**: 过滤掉 10-20% 的低质知识
