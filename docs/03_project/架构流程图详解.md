## 第二部分：用户消息处理流程

### 完整的请求-响应流程图

```
┌─────────────────────────────────────────────────────────────────┐
│  用户与 AI Child 的完整交互流程                                 │
└─────────────────────────────────────────────────────────────────┘

                        ┌──────────────────┐
                        │ 用户输入          │
                        │ (Telegram/        │
                        │  WebSocket/      │
                        │  Webhook)        │
                        └────────┬─────────┘
                                 │
                                 ▼
                        ┌──────────────────┐
                        │ Bot Bridge Layer  │
                        │ (接收并转换格式)  │
                        └────────┬─────────┘
                                 │
                                 ▼
                    ┌────────────────────────┐
                    │ FastAPI /chat/text     │
                    │ (或其他 API 端点)      │
                    │ Input: text, session   │
                    └────────┬───────────────┘
                             │
                             ▼
                    ╔════════════════════════╗
                    ║  STEP 1: PERSIST MSG   ║  ← 用户消息保存
                    ╚════════════┬═══════════╝
                                 │
                add_message(session, "user", text)
                        ↓
                INSERT INTO conversations
                        │
                        ▼
                    ╔════════════════════════╗
                    ║  STEP 2: BUILD CONTEXT ║  ← 上下文构建
                    ╚════════════┬═══════════╝
                                 │
        ┌────────────────────────┼────────────────────────┐
        │                        │                        │
        ▼                        ▼                        ▼
    get_ai_name()        get_or_create_profile()    get_recent_messages(20)
        ↓                        ↓                        ↓
    [name]              [is_sleeping flag]          [历史对话]
        │                        │                        │
        └────────────────────────┼────────────────────────┘
                                 │
                                 ▼
        ┌────────────────────────────────────────┐
        │ search_knowledge(user_text)            │
        │ get_all_knowledge()                    │
        │                                        │
        │ → 相关知识 + 全部知识               │
        │ → 最多 30-40 条知识项                │
        └─────────────┬──────────────────────────┘
                      │
                      ▼
    ╔═════════════════════════════════════════╗
    ║  STEP 3: BUILD MESSAGES FOR GPT        ║  ← 消息列表构建
    ╚═════════════════┬═══════════════════════╝
                      │
        messages = [
            {role: "system", content: _build_system_prompt(name, is_sleeping)},
            {role: "system", content: "Things you have learned:\n..."},
            ...历史消息...
            {role: "user", content: user_text}
        ]
                      │
                      ▼
    ╔═════════════════════════════════════════╗
    ║  STEP 4: LOAD TOOL DEFINITIONS         ║  ← 工具加载
    ╚═════════════════┬═══════════════════════╝
                      │
        tool_defs = [
            BUILTIN_TOOLS (web_search, execute_code, create_tool),
            ...get_all_tools()  (用户创建的工具)
        ]
                      │
                      ▼
    ╔═════════════════════════════════════════╗
    ║  STEP 5: FUNCTION CALLING LOOP         ║  ← 函数调用循环
    ║         (最多 10 轮)                    ║
    ╚═════════════════┬═══════════════════════╝
                      │
                      ▼
            ┌─────────────────────────┐
            │ response = await        │
            │ openai.chat.completions │
            │ .create(...)            │
            │                         │
            │ model="gpt-4o"          │
            │ messages=messages       │
            │ tools=tool_defs         │
            │ tool_choice="auto"      │
            └────────┬────────────────┘
                     │
                     ▼
            ┌─────────────────────────┐
            │ finish_reason == ?       │
            └────┬───────────┬────────┘
                 │           │
         "tool_calls"    "end"/"stop"
                 │           │
                 ▼           ▼
    ┌───────────────────┐   │
    │ 有工具调用！      │   │
    │                   │   │
    │ for tc in         │   ▼
    │   tool_calls:     │
    │                   │
    │ dispatch_tool()   │
    │ ├─ web_search()  │
    │ ├─ execute_code()│
    │ ├─ create_tool() │
    │ └─ user tools()  │
    │                   │
    │ 获得结果          │   
    │ ↓                 │
    │ messages.append() │
    │ (tool 消息)       │
    │                   │
    │ 继续循环 ↻        │
    └───────────────────┘
                     │
                     ▼
            ┌──────────────────────┐
            │ reply_text获得        │
            │ 最终文本回复         │
            │ (没有更多工具)       │
            └────────┬─────────────┘
                     │
                     ▼
    ╔══════════════════════════════════════╗
    ║  STEP 6: EXTRACT/GENERATE QUESTION  ║  ← 问题提取
    ╚══════════════════┬═══════════════════╝
                       │
        if "[QUESTION:" in reply_text:
            ↓
            提取 [QUESTION: xxx]
            ↓
            proactive_question = ...
        else:
            if turn_count % interval == 0:
                ↓
                _generate_proactive_question()
                ↓
                GPT-4o 生成新问题
                       │
                       ▼
    ╔══════════════════════════════════════╗
    ║  STEP 7: PERSIST ASSISTANT MESSAGE  ║  ← 回复保存
    ╚══════════════════┬═══════════════════╝
                       │
        add_message(
            session,
            role="assistant",
            content=reply_text
        )
                       │
                       ▼
            INSERT INTO conversations
                       │
                       ▼
    ╔══════════════════════════════════════╗
    ║  STEP 8: PERSIST QUESTION (if any)  ║  ← 问题保存
    ╚══════════════════┬═══════════════════╝
                       │
        if proactive_question:
            ↓
            add_pending_question(
                session,
                proactive_question
            )
                       │
                       ▼
            INSERT INTO pending_questions
                       │
                       ▼
    ╔══════════════════════════════════════╗
    ║  STEP 9: RETURN RESPONSE             ║  ← 返回响应
    ╚══════════════════┬═══════════════════╝
                       │
        {
            "reply": reply_text,
            "proactive_question": question
        }
                       │
                       ▼
            Bot Bridge 接收
                       │
                       ▼
        通过 Telegram/WebSocket
        发送给用户
                       │
                       ▼
                    ┌──────────────────┐
                    │  用户看到回复    │
                    │  和新问题        │
                    └──────────────────┘
```

### 睡眠决策树

```
系统时间检查 (每分钟)
    │
    ├─ 当前小时 == sleep_hour?
    │  │
    │  ├─ YES ──→ _had_event_today("sleep")?
    │  │        ├─ YES ─→ 跳过 (已睡过)
    │  │        └─ NO ──→ go_to_sleep()
    │  │                 ├─ Set is_sleeping=True
    │  │                 ├─ Create SleepEvent
    │  │                 ├─ 启动后台任务
    │  │                 │  consolidate_memories()
    │  │                 └─ 等待
    │  │
    │  └─ NO
    │
    ├─ 当前小时 == wake_hour?
    │  │
    │  ├─ YES ──→ _had_event_today("wake")?
    │  │        ├─ YES ─→ 跳过 (已醒过)
    │  │        └─ NO ──→ wake_up()
    │  │                 ├─ Set is_sleeping=False
    │  │                 ├─ Create SleepEvent
    │  │                 └─ 等待 Bot 推送
    │  │
    │  └─ NO
    │
    ├─ is_sleeping 状态变化?
    │  │
    │  ├─ YES ──→ 更新所有后续请求的系统提示
    │  │
    │  └─ NO
    │
    └─ 下一分钟再检查
```

## 第六部分：错误处理与恢复机制

### 系统级的错误处理流程

```
异常/错误发生
    │
    ├─ 错误类型?
    │
    ├─ OpenAI API 错误
    │  │
    │  ├─ Rate Limited?
    │  │  └─ 等待后重试
    │  │
    │  ├─ Invalid Token?
    │  │  └─ 日志记录，返回500
    │  │
    │  └─ 其他?
    │     └─ 日志记录，返回错误响应
    │
    ├─ 数据库错误
    │  │
    │  ├─ 连接失败?
    │  │  └─ 重新连接或返回错误
    │  │
    │  ├─ 约束违反?
    │  │  └─ Rollback + 日志
    │  │
    │  └─ 其他?
    │     └─ Rollback + 日志
    │
    ├─ 代码执行错误
    │  │
    │  ├─ 超时?
    │  │  └─ Kill process + 返回"[Timeout]"
    │  │
    │  ├─ 被安全检查阻止?
    │  │  └─ 返回"[Blocked] ..."
    │  │
    │  └─ 其他?
    │     └─ 返回 stderr 内容
    │
    └─ 网络错误
       │
       ├─ 搜索失败?
       │  └─ 返回 [] (空结果)
       │
       └─ 其他?
          └─ 返回错误消息
                      │
                      ▼
            用户看到:
            "抱歉，遇到了问题..."
            或部分功能降级

            后端:
            详细日志已记录
            服务继续运行
            下一次请求正常处理
```
