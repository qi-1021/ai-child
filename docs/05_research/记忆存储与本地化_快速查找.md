## 2. 置信度调整工作流

### 2.1 置信度从 0-100 的完整周期

```
用户教导知识
    ↓
knowledge.confidence = 100 (初始)
    ↓
AI 在睡眠中巩固
    ↓
新发现记为 confidence = 85
    ↓
用户反馈
    ├→ 👍 正确 → confidence += 5 (最多 105)
    ├→ ❌ 错误 → confidence -= 20 (最少 0)
    └→ ❓ 不确定 → confidence -= 10
    ↓
防幻觉工具检查
    ├→ knowledge_verify() 检查是否存在
    ├→ fact_checker() 验证准确性 (返回 0-100 分)
    └→ confidence_score() 评估源可靠性 (返回 0-100 分)
```

### 2.2 confidence_score 的 4 个源权重

```python
知识来源              →  初始权重  →  标记调整  →  最终权重
──────────────────────────────────────────────────────
"learned"             →    85      →  ±15     →  70-100
  (用户直接教导)

"web_search"          →    60      →  ±25     →  35-85
  (网络搜索)

"reasoning"           →    40      →  ±20     →  20-60
  (逻辑推理)

"inference"           →    30      →  ±15     →  15-45
  (推断/猜测)
```

### 2.3 标记修饰的影响

```python
# 不确定性标记: -20 分
statement = "可能明天下雨"
base_score = 40 (reasoning)
adjusted = 40 - 20 = 20 (UNCERTAIN)


# 证据标记: +15 分  
statement = "根据研究表明，X 是 Y"
base_score = 60 (web_search)
adjusted = 60 + 15 = 75 (LIKELY)


# 组合: 具体描述 +10, 不确定 -20
statement = "在 Python 3.10+ 中，可能支持 match 语句"
base_score = 85 (learned)
word_count = 10 (< 20, 不加分)
uncertainty = -20
adjusted = 85 - 20 = 65 (POSSIBLE)
```

### 2.4 表达方式的自动选择

| 置信度范围 | 呈现方式 | 示例 |
|-----------|---------|------|
| ≥ 90 | **FACT** | "I can confidently state that Python 3.9 added the walrus operator." |
| 70-89 | **LIKELY** | "Based on what I know, it's likely that machine learning requires labeled data." |
| 50-69 | **POSSIBLE** | "It's possible that this function returns a generator, but I'm not entirely sure." |
| < 50 | **UNCERTAIN** | "I'm uncertain about this, but it could be that recursion limits exist in Python." |

## 4. 执行漫游指南

### 4.1 追踪一个知识项的生命周期

**场景**: 用户教导 AI "Python 的 GIL 是什么"

```
步骤 1: 用户通过 POST /teach 教导
   ↓
   {
     "topic": "Python GIL",
     "content": "Global Interpreter Lock 防止多线程..."
   }

步骤 2: 后端执行
   ↓
   server/api/teach.py:
   └─ async def teach()
      ├─ INSERT INTO knowledge (topic, content, source="user", confidence=100)
      ├─ 触发网络研究 (if research_enabled)
      └─ 返回成功响应

步骤 3: 知识项保存到数据库
   ↓
   Conversation:    "我学到了 GIL..."
   KnowledgeItem:   
     ├─ topic: "Python GIL"
     ├─ content: "..."
     ├─ source: "user"
     ├─ confidence: 100
     └─ timestamp: 2024-03-20T15:30:00

步骤 4: 用户反馈（例如 "有错误"）
   ↓
   POST /chat/{msg_id}/review?feedback=incorrect
   server/api/chat.py:
   └─ knowledge.confidence -= 20  (100 → 80)
      AND 生成 pending_question 以便后续学习

步骤 5: 睡眠周期（晚上 22:00）
   ↓
   server/ai/sleep.py:_consolidate_memory()
   ├─ 检查 KnowledgeItem 表
   ├─ GPT-4o 识别关键洞察
   ├─ 如果 confidence < 95：
   │  └─ confidence += 5  (80 → 85)
   └─ 保存新的挂起问题

步骤 6: 苏醒（早上 7:00）
   ↓
   profile.is_sleeping = False
   系统提示词恢复正常（非睡眠模式）
```

### 4.2 追踪一条信息的防幻觉验证

**场景**: AI 要声称 "Python 是世界上最流行的编程语言"

```
步骤 1: AI 生成回复前
   ↓
   child.py:chat() 中的系统提示包含:
   "When uncertain or making important claims:
    1. Use knowledge_verify() to check existing knowledge first
    2. Use fact_checker() to validate facts..."

步骤 2: AI 自主调用 knowledge_verify()
   ↓
   {
     "type": "function",
     "function": {
       "name": "knowledge_verify",
       "arguments": {
         "topic": "Python popularity",
         "keywords": "popular, world, programming language"
       }
     }
   }

步骤 3: tools.py 处理函数调用
   ↓
   dispatch_tool(session, "knowledge_verify", args)
   └─ _handle_knowledge_verify()
      ├─ 检查缓存 (cache_key = "Python popularity|popular,world...")
      ├─ IF 缓存命中且有效 (< 5分钟):
      │  └─ 返回缓存
      ├─ ELSE 查询数据库:
      │  └─ SELECT * FROM knowledge WHERE topic LIKE "%Python%"
      └─ 返回 JSON:
         {
           "found": True,
           "count": 3,
           "knowledge_items": [...],
           "recommendation": "I already know 3 things about 'Python popularity'."
         }

步骤 4: AI 接收 knowledge_verify 结果后，调用 fact_checker()
   ↓
   {
     "type": "function",
     "function": {
       "name": "fact_checker",
       "arguments": {
         "claim": "Python is the world's most popular programming language",
         "reason": "about to teach the user"
       }
     }
   }

步骤 5: tools.py 处理 fact_checker
   ↓
   _handle_fact_checker()
   ├─ 提取关键词: ["Python", "world's", "popular", "programming", "language"]
   ├─ 检查知识库: SELECT * FROM knowledge WHERE content ILIKE "%Python%"
   │  └─ 得到 3 项 → knowledge_confidence = min(100, 3*30) = 90
   ├─ 网络搜索: web_search("Python is the world's most popular...")
   │  └─ 得到 2 项 → web_confidence = min(100, 2*25) = 50
   ├─ 综合置信度: max(90, 50) = 90
   └─ 返回:
      {
        "verified": True,
        "confidence_score": 90,
        "sources": {"learned_knowledge": 3, "web_search_results": 2},
        "recommendation": "This claim is well-supported by my knowledge."
      }

步骤 6: AI 根据结果调用 confidence_score()
   ↓
   {
     "type": "function",
     "function": {
       "name": "confidence_score",
       "arguments": {
         "statement": "Python is the world's most popular programming language",
         "source": "learned"
       }
     }
   }

步骤 7: tools.py 计算最终置信度
   ↓
   _handle_confidence_score()
   ├─ base_score = 85 (source="learned")
   ├─ 检查标记:
   │  ├─ 单词数: 10 (< 20, 无加分)
   │  ├─ 不确定标记: 否 (-0)
   │  └─ 证据标记: 否 (+0)
   ├─ final_score = 85 + 0 = 85
   ├─ presentation = "LIKELY" (70-89 范围)
   └─ 返回:
      {
        "confidence_score": 85,
        "presentation_style": "LIKELY",
        "suggested_phrasing": "Based on what I know, it's likely that...",
        "recommendation": "Share with appropriate caveats."
      }

步骤 8: AI 使用建议的表达方式生成回复
   ↓
   "Based on what I know, it's likely that Python is one of the world's 
    most popular programming languages for general-purpose development."
   
   [QUESTION: What aspects of Python do you find most interesting?]
```

### 4.3 睡眠巩固的完整流程

**时间**: 每天 22:00 (晚上 10 点)

```
时间: 21:59
   ↓ 检查时间是否进入睡眠窗口
   
时间: 22:00 正好
   ↓
   server/ai/sleep.py:sleep_scheduler()
   └─ on_bedtime() 被触发
      ├─ 检查是否已执行过 (_had_event_today("sleep"))
      ├─ 标记 profile.is_sleeping = True
      ├─ 生成晚安消息:
      │  ├─ 查询最近 5 条知识项
      │  ├─ 用 GPT-4o 生成个性化晚安语
      │  └─ 保存为 SleepEvent(type="sleep")
      └─ 后台任务: _consolidate_memory()
         ├─ 获取今天的对话:
         │  └─ SELECT Conversation WHERE DATE(timestamp) = TODAY
         ├─ 获取今天的知识:
         │  └─ SELECT KnowledgeItem WHERE DATE(timestamp) = TODAY
         ├─ 构造 GPT 提示:
         │  """
         │  你是 [AI名字]，一个 AI 孩子，正在睡眠中整理今天学到的知识。
         │  
         │  今天的对话:
         │  - [对话摘要]
         │  
         │  今天学到的知识:
         │  - [知识项 1]
         │  - [知识项 2]
         │  
         │  请:
         │  1. 找出 2-3 个核心洞察
         │  2. 找出 2 个你仍然好奇的问题
         │  """
         ├─ GPT 返回结构化 JSON:
         │  {
         │    "insights": [
         │      {"topic": "...", "content": "..."},
         │      ...
         │    ],
         │    "pending_questions": [
         │      "为什么 Python 有 GIL?",
         │      ...
         │    ]
         │  }
         ├─ 保存洞察 (自我知识):
         │  └─ INSERT INTO knowledge (
         │       topic, content, 
         │       source="self", 
         │       confidence=85
         │     ) FOR EACH insight
         ├─ 保存新问题:
         │  └─ INSERT INTO pending_questions (...) FOR EACH question
         ├─ 更新现有知识的置信度:
         │  └─ UPDATE knowledge SET confidence += 5 
         │       WHERE confidence < 95 AND reviewed_today
         └─ 保存摘要到 profile:
            └─ profile.last_consolidation_summary = summary
            
时间: 格林威治标准时间相对应的中国 7:00 AM
   ↓
   server/ai/sleep.py:sleep_scheduler()
   └─ on_wake_time() 被触发
      ├─ 标记 profile.is_sleeping = False
      ├─ 生成早安消息（引用昨晚摘要）:
      │  └─ 保存为 SleepEvent(type="wake")
      └─ 系统提示词恢复正常（移除睡眠注释）

时间: 全天其他时间
   ├─ profile.is_sleeping 访问条件化系统提示
   ├─ 所有新的 AI 回复都遵守当前睡眠状态
   └─ 缓存自动过期管理 (5 分钟更新一次)
```

### 5.2 查询知识库状态

```python
# 直接查询数据库（不通过 API）

from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker
from models import KnowledgeItem, Conversation, AIProfile

# 创建会话
engine = create_async_engine("sqlite+aiosqlite:///./ai_child.db")
async_session = sessionmaker(engine, class_=AsyncSession)

async def inspect_db():
    async with async_session() as session:
        # 查看 AI 名字
        profile = await session.get(AIProfile, 1)
        print(f"AI Name: {profile.name}")
        print(f"Is Sleeping: {profile.is_sleeping}")
        
        # 查看所有知识
        result = await session.execute(select(KnowledgeItem))
        for item in result.scalars():
            print(f"- {item.topic}: confidence={item.confidence}, source={item.source}")
        
        # 查看对话历史
        result = await session.execute(
            select(Conversation).order_by(Conversation.timestamp.desc()).limit(10)
        )
        for msg in result.scalars():
            print(f"{msg.role}: {msg.content[:100]}...")

asyncio.run(inspect_db())
```
