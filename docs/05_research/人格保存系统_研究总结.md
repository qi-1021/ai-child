## 🌙 睡眠周期中的人格变化

```
每天早上 07:00 (唤醒)
┌─ [sleep.py] sleep_scheduler 触发 "wake" 事件
├─ AIProfile.is_sleeping = False
├─ sys_prompt 从 sleep_tone 切换为 normal_tone
├─ 生成早安消息（参考 last_consolidation_summary）
└─ 创建 SleepEvent(type="wake", message="...")


一整天 07:00 - 22:00 (清醒状态)
├─ AIProfile.is_sleeping = False
├─ 正常对话
├─ 积累知识 → KnowledgeItem 表
└─ 生成问卷 → PendingQuestion 表


每天晚上 22:00 (睡眠)
┌─ [sleep.py] sleep_scheduler 触发 "sleep" 事件
├─ AIProfile.is_sleeping = True
├─ sys_prompt 切换为 sleep_tone (困倦/慵懒)
│
├─ 后台任务: consolidate_memories()
│  ├─ 查询今日所有知识: SELECT * FROM knowledge WHERE date(timestamp) = today
│  ├─ 使用 GPT-4o 总结: "今天学到关于 [topic1], [topic2], [topic3] 的知识，关键要点是..."
│  ├─ AIProfile.last_consolidation_summary = "..." (更新)
│  ├─ 生成新问卷: "明天想学关于...?"
│  └─ 存储为 high-confidence 知识
│
├─ 创建 SleepEvent(type="sleep", message="晚安，今天学到很多呢...")
└─ 用户看到个性化睡眠消息
```

## 💡 架构亮点分析

### 亮点 1: 单例模式确保唯一性

```python
# ✅ 好的设计
class AIProfile(Base):
    __tablename__ = "ai_profile"
    id = Column(Integer, primary_key=True, default=1)  # 固定为 1

async def get_or_create_profile(session) -> AIProfile:
    result = await session.execute(
        select(AIProfile).where(AIProfile.id == 1)  # 永远是同一行
    )
    profile = result.scalar_one_or_none()
    if profile is None:
        profile = AIProfile(id=1, name=None)  # 首次自动创建
    return profile
```

**优点**：
- 无需用户ID即可全局访问
- 避免多个 Profile 行的混乱
- 代码简洁

### 亮点 2: GPT-4o 自然语言提取

```python
# 用户可能的回答:
# "就叫小明吧" → "小明"
# "你叫 Alex" → "Alex"
# "Tom sounds nice" → "Tom"

async def extract_name_from_answer(answer: str) -> str:
    prompt = f"Extract the name from: {answer}"
    response = await _client.chat.completions.create(...)
    return response.choices[0].message.content
```

**优点**：
- 支持自然表达（无需特定格式）
- 支持多语言（中英混合）
- 错误降级（如果 GPT 失败则截断原始答案）
