## 1. 记忆分类机制

### 1.1 知识库置信度系统

**主文件**: [server/models/__init__.py](server/models/__init__.py)

```python
class KnowledgeItem(Base):
    """Explicit knowledge taught by the user."""
    __tablename__ = "knowledge"
    
    id = Column(Integer, primary_key=True, index=True)
    topic = Column(String(256), nullable=False, index=True)
    content = Column(Text, nullable=False)
    source = Column(String(64), default="user")  # "user" | "self"
    confidence = Column(Integer, default=100)     # ✓ 0-100 置信度系统
    timestamp = Column(DateTime(timezone=True), default=_utcnow)
    last_reviewed = Column(DateTime(timezone=True), nullable=True)
```

**关键点**:
- 置信度范围: **0-100**（整数）
- 默认值: **100**（用户直接教导）
- 来源标记: `source` 阻断 `"user"` 或 `"self"`（自我发现）

### 1.2 对话历史（无优先级标记）

**主文件**: [server/models/__init__.py](server/models/__init__.py)

```python
class Conversation(Base):
    """A single conversation turn between user and AI child."""
    __tablename__ = "conversations"
    
    id = Column(Integer, primary_key=True, index=True)
    role = Column(String(16), nullable=False)     # "user" | "assistant"
    content = Column(Text, nullable=False)
    content_type = Column(String(32), default="text")  # text | image | audio | mixed
    media_path = Column(String(512), nullable=True)
    timestamp = Column(DateTime(timezone=True), default=_utcnow)
    metadata_ = Column("metadata", JSON, default=dict)  # 灵活的元数据
```

**说明**:
- ❌ 对话层面**无优先级字段**
- 支持 JSON 元数据扩展，可用于标记重要性
- 区分媒体类型（文本/图像/音频/混合）

### 1.3 待回答问题（隐含重要性）

**主文件**: [server/models/__init__.py](server/models/__init__.py)

```python
class PendingQuestion(Base):
    """Questions the AI child wants to ask the user."""
    __tablename__ = "pending_questions"
    
    id = Column(Integer, primary_key=True, index=True)
    question = Column(Text, nullable=False)
    topic = Column(String(256), nullable=True)
    answered = Column(Boolean, default=False)

    created_at = Column(DateTime(timezone=True), default=_utcnow)

```

**重要性隐含机制**:
- 由 **sleep.py** 在睡眠时生成关键问题（见 [1.4](#14-睡眠周期的知识巩固)）
- 名字问题使用特殊标记: `topic = "__name__"` —— **最高优先级**

### 1.4 睡眠周期的知识巩固

**主文件**: [server/ai/sleep.py](server/ai/sleep.py)

知识巩固流程（记忆重要性评价）:

```python
# 睡眠时执行
async def _consolidate_memory(session: AsyncSession, name: str | None):
    """
    1. 审查今天的对话和知识项
    2. GPT-4o 提取关键洞察
    3. 保存为高置信度自我知识 (confidence=85%)
    4. 生成 1-2 个新的好奇问题
    5. 轻微提高强化知识的置信度
    """
```

**置信度调整规则** （来自文档和代码）:

| 事件 | 置信度变化 |
|------|----------|
| 用户直接教导 | `confidence = 100` |
| 自我发现/睡眠巩固 | `confidence = 85` |
| 用户确认 👍 | `confidence += 5` |
| 用户指正 ❌ | `confidence -= 20` |
| 用户不确定 ❓ | `confidence -= 10` |

### 2.2 系统提示词的定义

**主文件**: [server/ai/child.py](server/ai/child.py)

#### 系统提示生成函数

```python
def _build_system_prompt(name: str | None, is_sleeping: bool = False) -> str:
    """动态构建系统提示 — 反映 AI 当前身份和睡眠状态"""
    
    # 身份部分（支持中英文）
    if name is None:
        identity = (
            "You are a newly born AI child — you don't have a name yet. "
            "You must ask the person you are talking to what you should be called "
            "before doing anything else."
        )
    else:
        identity = f"You are {name}, an AI child."
    
    # 睡眠状态部分（仅英文）
    if is_sleeping:
        sleep_note = (
            "\n\nNote: You are currently in rest/sleep mode. "
            "Respond briefly and gently — you are half-asleep..."
        )
    else:
        sleep_note = ""
```

**特点**:
- ❌ 系统提示**主体是英文**（不支持中文系统提示）
- ✓ 支持参数化的 AI 名字
- ✓ 支持睡眠/清醒状态的动态切换
- ⚠️ 需要改进：系统提示可用 `language` 参数支持中文

## 3. 数据存储结构

### 3.1 完整数据库表设计

**主文件**: [server/models/__init__.py](server/models/__init__.py)

#### 表 1: conversations（对话历史）

```python
class Conversation(Base):
    __tablename__ = "conversations"
    id              INT PRIMARY KEY
    role            STRING(16)           # "user" | "assistant"
    content         TEXT
    content_type    STRING(32) DEFAULT   # "text", "image", "audio", "mixed"
    media_path      STRING(512) NULL
    timestamp       DATETIME DEFAULT now()
    metadata_       JSON DEFAULT {}      # 灵活扩展
```

**用途**: 存储完整对话历史
**索引**: `id`, `role`, `timestamp` 隐含

#### 表 2: knowledge（知识库）

```python
class KnowledgeItem(Base):
    __tablename__ = "knowledge"
    id              INT PRIMARY KEY
    topic           STRING(256) NOT NULL INDEXED
    content         TEXT NOT NULL
    source          STRING(64) DEFAULT "user"  # "user" | "self"
    confidence      INT DEFAULT 100             # 0-100 置信度
    timestamp       DATETIME DEFAULT now()
    last_reviewed   DATETIME NULL
```

**索引**: `topic` （优化 search）
**键字段**:
- `confidence`: 知识根可信度
- `source`: 来源追溯

#### 表 3: pending_questions（AI 的问题）

```python
class PendingQuestion(Base):
    __tablename__ = "pending_questions"
    id              INT PRIMARY KEY
    question        TEXT NOT NULL
    topic           STRING(256) NULL       # "__name__" 标记命名问题
    answered        BOOLEAN DEFAULT False
    answer          TEXT NULL
    created_at      DATETIME DEFAULT now()
    answered_at     DATETIME NULL
```

**特殊处理**:
- 名字问题: `topic = "__name__"` （最高优先级）
- 睡眠巩固时生成（见 sleep.py）

#### 表 4: ai_profile（个性数据 — 分离存储）

```python
class AIProfile(Base):
    """Singleton row (id=1) 持有 AI 的个性数据"""
    __tablename__ = "ai_profile"
    
    id                          INT PRIMARY KEY = 1  # 单行单例
    name                        STRING(128) NULL     # 初始为 NULL
    created_at                  DATETIME DEFAULT now()
    named_at                    DATETIME NULL        # 命名时间戳
    is_sleeping                 BOOLEAN DEFAULT False
    last_consolidation_summary  TEXT NULL            # 睡眠巩固摘要
```

**设计特点**:
- ✓ **分离的"个性数据"表** —— 与对话、知识分开
- ✓ Singleton 模式（`id=1`）
- ✓ 支持睡眠状态
- 缺失字段: 没有 `preferred_language`、`personality_traits` 等

#### 表 5: tools（自创工具）

```python
class Tool(Base):
    __tablename__ = "tools"
    id                  INT PRIMARY KEY
    name                STRING(128) NOT NULL UNIQUE INDEXED
    description         TEXT NOT NULL
    code                TEXT NOT NULL
    parameters_schema   JSON NOT NULL DEFAULT {}  # OpenAI 格式
```

#### 表 6: sleep_events（睡眠事件）

```python
class SleepEvent(Base):
```

### 3.2 缓存策略

**主文件**: [server/ai/tools.py](server/ai/tools.py#L28-29)

```python
# 防幻觉工具缓存（避免重复查询）
_knowledge_cache: Dict[str, tuple] = {}      # {cache_key: (timestamp, data)}
_confidence_cache: Dict[str, float] = {}
```

#### 缓存实现（knowledge_verify）

```python
# 检查缓存
cache_key = f"{topic}|{keywords}"
if cache_key in _knowledge_cache:
    cached_time, cached_data = _knowledge_cache[cache_key]
    import time
    if time.time() - cached_time < 300:  # ✓ 5 分钟缓存有效期
        return json.dumps(cached_data)

# ... 执行查询后保存到缓存 ...
_knowledge_cache[cache_key] = (__import__("time").time(), result_data)
```

**缓存策略**:
- **类型**: In-memory 字典（不持久化）
- **有效期**: 5 分钟（300 秒）
- **键**: `f"{topic}|{keywords}"`
- **值**: `(timestamp, result_data)` 元组

**缺点**:
- ❌ 服务器重启后丧失
- ❌ 多进程环境下失效
- ⚠️ 适合开发/演示，生产需 Redis

### 3.3 数据关联关系

```
AI Child Learning Flow
├─ AIProfile (单例，id=1)_________
│  ├─ name: 中文或英文名字
│  ├─ is_sleeping: 睡眠状态
│  └─ last_consolidation_summary: 摘要
│
├─ Conversations (完整历史)
│  ├─ user: 用户输入
│  └─ assistant: AI 回复
│
├─ KnowledgeItem (知识库)
│  ├─ source: "user" (教导) | "self" (自我发现)
│  └─ confidence: 0-100 (置信度)
│
├─ PendingQuestion (好奇心驱动)
│  ├─ topic: "__name__" | 其他主题
│  └─ answered: 是否回答
│
├─ Tools (自创工具)
│  └─ code: 持久化 Python 函数
│
└─ SleepEvent (睡眠周期事件)
   ├─ sleep: 入睡时生成
   └─ wake: 睡醒时生成
```
